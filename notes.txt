REQUIREMENT
* purchase voucher for a phone number
* voucher can be displayed on web page, or sent via sms
* display purchase history for a phone number
NFR
* voucher generating should complete within 30 seconds
* support for multiple voucher vendors

MICROSERVICES
* prepaid-sim-service
- calls voucher-provisioning-service to generate voucher
- manage purchase history

* voucher-provision-service
- encapsulates the logic for interacting with 3rd party apis
- expects caller (order-service) to store result. This service itself is stateless - it does not interact with database

* notification-service
- send sms to customer

FLOWS
* order voucher
user -> order-service -> voucher-provision-service -> 3rd party api

* display order history
user (phone + otp) -> order-service -> database

DATABASE TABLES
* voucher_order
- id
- phone_number
- voucher_id
- voucher_value
- voucher_telco
- voucher_code
- created_date
- status
